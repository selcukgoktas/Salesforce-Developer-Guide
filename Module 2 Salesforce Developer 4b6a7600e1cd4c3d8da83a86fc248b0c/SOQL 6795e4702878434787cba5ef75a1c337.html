Definition

SOQL is a language that gets record data from a Salesforce database. In¬†Object-Oriented Programming for Admins ¬†you learned that you can use Data Manipulation Language (DML) statements to insert, update, and delete records. But what if you want to get data that already exists in the database? SOQL makes it simple to retrieve data.

Creating SOQL queries is similar to creating a report. When you create a report, you answer questions such as

  • What fields do I want to display?
  • Where are those fields located?

//SOQL Query is not SQL 

SELECT Name, Email FROM Contact WHERE Title='Clarusway'

// required clauses SELECT * FROM

WHERE

  • AND 
    WHERE FirstName = 'Stella' AND LastName = 'Pavlova'
// Use AND to return records that meet two conditions.
  • OR
    WHERE LastName = 'James' OR LastName = 'Barr'
// Use OR to return records that meet one of two conditions.
  • IN
    WHERE LastName IN ('James', 'Barr', 'Nedaerk', 'Forbes')
// Use IN to return records that meet at least one of three or more conditions.

LIMIT

The¬†LIMIT¬†keyword sets the maximum number of records to return.

SELECT Email, Name FROM Contact LIMIT 5

ORDER BY

  • ASC
    SELECT Name, Email FROM Contact
ORDER BY Name ASC
LIMIT 5

// ascending order
  • DESC
    SELECT Name, Email FROM Contact
ORDER BY Email DESC
LIMIT 5

// descending order
  • NULLS FIRST | LAST
    SELECT Name, Email FROM Contact
ORDER BY Email
NULLS LAST

// Returns null records at the beginning

Aggregate Functions

In reports, you use roll-up summary fields to calculate values from related records. For example, you might want to calculate the total number of properties, or the average or highest value of properties in a city. In SOQL, you use aggregate functions to perform those and other calculations.

SELECT COUNT(Name) FROM Broker__c
// Returns the number of rows that are associated with the field

SELECT COUNT_DISTINCT(City__c) FROM Property__c
// Returns the number of unique rows that match the query criteria

SELECT MIN(Days_On_Market__c) FROM Property__c
SELECT MAX(Beds__c)FROM Property__c
// minimum and maximum value field

SELECT City__c, AVG(Days_On_Market__c) FROM Property__c GROUP BY City__c
// Returns the average value of a numeric field

SELECT SUM(Price__c), Broker__r.Name FROM Property__c GROUP BY Broker__r.Name
// Returns the total value of a numeric field
  • GROUP BY will group the results according to the preferences.

Relational Queries

Identification of the Scheme

As an admin, you‚Äôre familiar with master-detail relationships in an org. A master-detail relationship is a parent-to-child relationship. The master object is the parent, and the detail object is the child. If the two objects you want to include in a query have a master-detail relationship, then you know which is the parent and which is the child.

Child to Parent


// standard objects child to parent
SELECT Name, Account.Name FROM Contact

// custom objects child to parent
SELECT Address__c, Picture__c, Broker__r.Name FROM Property__c

Parent to Child

SELECT Name, (SELECT Name FROM Contacts) FROM Account

// filter the results
SELECT Name, (SELECT Name FROM Contacts) FROM Account 
WHERE Id IN (SELECT AccountId FROM Contact WHERE LastName = 'Forbes')


// parent to child subquery
SELECT Name, (SELECT Address__c, Price__c FROM Properties__r) FROM Broker__c
  • Use Plural Version of Child Standard Object ( You can see on Child Relationship Name Section
  • For Custom object you need to use __r extention
