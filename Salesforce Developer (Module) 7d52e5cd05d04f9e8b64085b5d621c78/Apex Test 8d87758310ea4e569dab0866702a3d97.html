Test runner

Unit test is a function that executed by a software called a test-runner

To deploy your code in production org, you need at least 75% code coverage.

While writing test

  • A short explanation on what you are testing
  • A part of your software that you want to test ( function or a class)
  • Test data to supply as input or parameter to your code
  • A result that you expect your code to produce

What to Test?

  • Negative Behaviour

Assert Methods

Use assert methods to prove that your code is behaving properly.

Testing is not about code coverage, it is about checking your behaviour and proving that your code works fine.

System.assert(x==y); // boolean
System.assert(x==y,'X is not equal to Y');

System.assertEquals(x,y); // not boolean
System.assertEquals(x,y,'X is not equal to Y');

System.assertNotEquals(x,y);
System.assertNotEquals(x,y,'X is Equal to Y');
