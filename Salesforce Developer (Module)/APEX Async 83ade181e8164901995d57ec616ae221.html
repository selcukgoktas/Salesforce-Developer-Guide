APEX Async

Future

Official Documentation : https://developer.salesforce.com/docs/atlas.en-us.200.0.apexcode.meta/apexcode/apex_async_overview.htm

  • Runs in its own thread and transactions
  • Executes when system resources are available.
  • It is not a realtime.
  • Must be static
  • Must have void return type
  • Parameters must be primitive or collection of primitive datatypes.
  • Future methods cannot be called from another future method.
  • Limit to 50 future calls per transaction
  • Can’t trigger future call in a future call
  • Use System.isFuture in code to avoid call a future method.

global class myClass {

		@future
		public static void myFutureMethod(){

public with sharing class LeadAsync {
    
    
    
    @future
    public static void methodName(Set<Id> exRecIds) {
        
        List<Lead> multiLead = [SELECT Id, LeadSource FROM Lead WHERE id IN :exRecIds];
        for (Lead rec : multiLead) {
            rec.LeadSource = 'Web';
        }
        
        update multiLead;
        
        
    }
}
public class LeadBatch implements Database.Batchable<SObject> {
    
    public Database.QueryLocator start(Database.BatchableContext BC){
        return Database.getQueryLocator('SELECT id FROM Lead');
    }
    
    public void execute(Database.BatchableContext BC, List<Lead> scope){
        for(Lead s:scope){
           s.LeadSource='Other';            
        }
        update scope;
    }
    
    public void finish(Database.BatchableContext BC){
     System.debug('Batch Finished');   
    }
}

Batch

  • Millions of Records
  • It divides your records into smaller batches and It process them separately.
  • long running process
  • can be scheduled https://developer.salesforce.com/docs/atlas.en-us.apexcode.meta/apexcode/apex_scheduler.htm
  • Implement database.batchable interface
  • operates over small batches chunks of records
  • Start / Execute / Finish Methods

  • Start
    • Executes only once
    • Return batch scope or records
    • can return up to 50 million records
  • Execute
    • Executes for each batch / chunk of records
    • Batch size 2000 ( 0-2000 Default value is 200 )
    • must have void return type
    • Each execution will get new set of governor limits

  • Finish
    • Execute only once
    • Called after all batches are processed
    • can be used for post processing

Queueable

  • Advanced version of Future Async Apex method.
  • You can directly pass Sobject which future cannot do this.
  • It allows you to monitor your apex jobs.
  • Unlike Future it allows you to chain your jobs.
  • Requires a separate class
  • 50 Jobs in a synchronous transaction but only 1 while in asynchronous transaction
  • Can chain queueables together
  • No limit on how many can be chained together
  • Invoke one by using system.enqueueJob()
public class LeadQueueable implements Queueable {

    public void execute(QueueableContext context){
          List<Lead> multiLead = [SELECT Id, LeadSource FROM Lead];
        
        for (Lead single:multiLead){
            single.LeadSource='Web';
        }
        update multiLead;
    }
}
LeadQueueable LQ = New LeadQueueable();


ID JobId=System.enqueueJob(LQ);

Scheduled

  • Requires a separate class
  • Schedule to a specific time
  • System.Schedule
  • Limit of 100 scheduled jobs
  • No callouts from scheduled apex call future or queueble methods.

public class LeadSchedulable implements Schedulable {

    public void execute(SchedulableContext sc){
          List<Lead> multiLead = [SELECT Id, LeadSource FROM Lead];
        
        for (Lead single:multiLead){
            single.LeadSource='Web';
        }
        update multiLead;
    }
}
LeadSchedulable LS = New LeadSchedulable();

String CronStr='00 00 1 ? * SAT';
